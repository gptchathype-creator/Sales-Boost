# Sales Boost Bot

Telegram bot для оценки экспертизы менеджеров по продажам через текстовый диалог-тест с клиентом. Система использует LLM для автоматической оценки ответов и предоставляет аналитику для администраторов.

## Возможности

- **Ролевой доступ**: менеджеры и администраторы
- **Диалог-тест**: пошаговый сценарий с сообщениями от "клиента"
- **Таймаут на ответы**: автоматический пропуск вопроса, если ответ не получен в течение 1 минуты
- **LLM-оценка**: автоматическая оценка ответов с детальной обратной связью
- **Mini App админ-панель**: удобный веб-интерфейс для просмотра результатов
- **Аналитика**: просмотр результатов, сводка по команде, экспорт в CSV
- **Уровни**: Junior/Middle/Senior на основе баллов

## Технологии

- Node.js 20 + TypeScript
- Telegraf (Telegram Bot Framework)
- Express.js (веб-сервер для Mini App)
- Prisma ORM + SQLite
- OpenAI API (GPT-4o mini для виртуального клиента и gpt-4o-mini-transcribe / Whisper для STT)
- Zod для валидации
- Telegram Mini App API

## Установка

### 1. Клонирование и установка зависимостей

```bash
npm install
```

### 2. Настройка окружения

Создайте файл `.env` на основе `.env.example`:

```bash
BOT_TOKEN=your_bot_token_here
OPENAI_API_KEY=your_openai_api_key_here
ADMIN_TELEGRAM_IDS=123456789,@username1,987654321
DATABASE_URL="file:./dev.db"
PORT=3000
MINI_APP_URL=http://localhost:3000
```

**Как получить BOT_TOKEN:**
1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot` и следуйте инструкциям
3. Скопируйте полученный токен в `.env`

**Как получить OPENAI_API_KEY:**
1. Зарегистрируйтесь на [OpenAI Platform](https://platform.openai.com/)
2. Создайте API ключ в разделе API Keys
3. Скопируйте ключ в `.env`

**ADMIN_TELEGRAM_IDS:**
- Можно использовать Telegram ID или username (с @ или без)
- Примеры:
  - `ADMIN_TELEGRAM_IDS=123456789,987654321` (только ID)
  - `ADMIN_TELEGRAM_IDS=@username1,@username2` (только username)
  - `ADMIN_TELEGRAM_IDS=123456789,@username1,987654321` (смешанный формат)
- Узнать свой Telegram ID можно через [@userinfobot](https://t.me/userinfobot)
- Username можно найти в настройках Telegram (Settings → Username)

### 3. Инициализация базы данных

```bash
npm i
npx prisma migrate dev
npm run dev
```

(Опционально: `npx prisma generate`, `npm run prisma:seed` для начального теста.)

### 4. Запуск

```bash
npm run dev
```

Для production:

```bash
npm run build
npm start
```

## Использование

### Для менеджеров

1. Откройте бота в Telegram
2. Отправьте `/start`
3. Укажите ваше полное имя (при первом запуске)
4. Нажмите "Начать тренировку"
5. Отвечайте на сообщения от "клиента" **текстом или голосовыми сообщениями** (бот распознаёт речь и использует текстовую транскрипцию в диалоге)
6. После завершения получите результаты

**Команды:**
- `/start` - начать работу
- `/check_admin` - проверить статус администратора (показывает ваш ID и username)
- `/delete_me` - удалить свой профиль и все данные

### Для администраторов

**Команды:**
- `/admin` - открыть админ-панель (с кнопкой Mini App)
- `/export_csv` - экспортировать все попытки в CSV

**Админ-панель:**
- **📱 Mini App** - веб-интерфейс с удобной навигацией (рекомендуется)
- **Последние попытки** - просмотр последних 20 завершенных тестов
- **По менеджерам** - выбор менеджера и просмотр его попыток
- **Сводка по команде** - агрегированная статистика

**Mini App функции:**
- Интерактивный интерфейс с вкладками
- Просмотр всех попыток с пагинацией
- Детальная информация по каждому менеджеру
- Статистика и сводка по команде
- Адаптивный дизайн для мобильных устройств

**Mini App:**
- Для HTTPS: открывается прямо в Telegram (webApp кнопка)
- Для HTTP (локальная разработка): открывается в браузере (URL кнопка)
- Никаких внешних сервисов не требуется!

## Диалог-движок (MVP0)

Тренировка с виртуальным клиентом (LLM) на основе данных одного авто из `data/car.json`:

- **Start training** — загрузка `data/car.json`, создание сессии, первый ответ «клиента» от LLM.
- Диалог: менеджер пишет текст → LLM генерирует ответ клиента → сохраняются в `DialogMessage`.
- Состояние (stage, checklist, client_turns) хранится в `TrainingSession.stateJson`.
- Завершение: визит согласован / контакт согласован / 14 ходов / менеджер не помогает.

Файлы: `data/car.json`, `src/data/carLoader.ts`, `src/state/defaultState.ts`, `src/llm/virtualClient.ts`, `src/handlers/training.ts`.

## Модель данных

- **User** - пользователи (менеджеры и администраторы)
- **Test** - тесты
- **TestStep** - шаги теста (сообщения от клиента)
- **Attempt** - попытка прохождения теста
- **AttemptAnswer** - ответ на конкретный шаг

## Критерии оценки

Система оценивает ответы по следующим критериям (0-5 баллов):

- **STRUCTURE** - структура и логика ответа
- **NEEDS_DISCOVERY** - выявление потребностей клиента
- **EMPATHY_TONE** - эмпатия и тон общения
- **OBJECTION_HANDLING** - работа с возражениями
- **VALUE_ARGUMENTATION** - аргументация ценности
- **NEXT_STEP_CTA** - призыв к следующему шагу
- **RISK_PHRASES** - избегание рискованных фраз

## Уровни

- **Junior**: 0-39 баллов
- **Middle**: 40-69 баллов
- **Senior**: 70-100 баллов

## Особенности

### Таймаут на ответы
- Каждый вопрос имеет ограничение по времени: **1 минута**
- Если ответ не получен в течение минуты, вопрос автоматически пропускается
- Пропущенные ответы сохраняются как `[Пропущено - превышено время ответа]`
- Тест продолжается со следующего вопроса

### Mini App
- Веб-интерфейс доступен через кнопку в админ-панели
- Работает на порту 3000 (настраивается через `PORT` в `.env`)
- Для production необходимо настроить `MINI_APP_URL` на публичный домен
- Использует Telegram Web App API для авторизации

## Безопасность

- Администраторы определяются через whitelist в `.env`
- Данные хранятся локально в SQLite
- Пользователи могут удалить свои данные через `/delete_me`
- Полные ответы не логируются в консоль
- Mini App проверяет авторизацию через Telegram initData

## Troubleshooting

**Бот не отвечает:**
- Проверьте правильность `BOT_TOKEN` в `.env`
- Убедитесь, что бот запущен (`npm run dev`)

**Ошибка оценки:**
- Проверьте `OPENAI_API_KEY` в `.env`
- Убедитесь, что у вас есть доступ к GPT-4 Turbo
- Проверьте баланс на OpenAI аккаунте

**Ошибка базы данных:**
- Удалите `dev.db` и запустите миграции заново
- Проверьте права доступа к файлу базы данных
